# Магма (ГОСТ Р 34.12-2015)

Реализация симметричного блочного шифра **«Магма»** на языке **Python** для практической работы по дисциплине **«Криптографические методы защиты информации»**.

## Закирова Екатерина Евгеньевна

## Описание проекта

Программа реализует шифрование и расшифрование данных с использованием алгоритма **Магма** и поддерживает режимы работы:

* **ECB** (Electronic Codebook);
* **CBC** (Cipher Block Chaining).

Особенность текущей версии программы заключается в том, что она:

* принимает на вход **любой файл** при шифровании;
* автоматически считывает файл как **байты**;
* преобразует данные в **hex** внутри программы;
* шифрует полученные данные;
* выводит в консоль исходное содержимое, hex-представление, шифртекст и результат расшифрования;
* сохраняет результат в отдельный файл.

## Возможности

* чтение любого входного файла в режиме шифрования;
* чтение hex-файла в режиме расшифрования;
* ввод секретного ключа через аргументы командной строки;
* выбор режима работы блочного шифра: `ecb` или `cbc`;
* использование вектора инициализации `IV` для режима `CBC`;
* автоматическое добавление и удаление `PKCS#7 padding`;
* вывод промежуточных данных в консоль;
* сохранение результата в новый файл.

## Требования

* Python 3.x
* стандартные библиотеки Python

Дополнительные криптографические библиотеки не требуются.

## Структура проекта

* `magma.py` — основная программа шифрования и расшифрования;
* `input*.txt` — входные файлы;
* `output*.txt` — файлы с результатом;
* `decoded*.txt` — файлы после расшифрования.

## Запуск программы

Общий формат запуска:

```bash
python magma.py encrypt <mode> <input_file> <output_file> <key_hex> [iv_hex]
python magma.py decrypt <mode> <input_file> <output_file> <key_hex> [iv_hex]
```

### Параметры

* `encrypt` — зашифрование;
* `decrypt` — расшифрование;
* `mode` — режим работы: `ecb` или `cbc`;
* `input_file` — входной файл;
* `output_file` — файл для записи результата;
* `key_hex` — ключ длиной **64 hex-символа** (256 бит);
* `iv_hex` — вектор инициализации для `CBC`, длиной **16 hex-символов** (64 бита).

## Примеры

### Шифрование в режиме ECB

```bash
python magma.py encrypt ecb input.txt output.txt ffeeddccbbaa99887766554433221100f0f1f2f3f4f5f6f7f8f9fafbfcfdfeff
```

### Расшифрование в режиме ECB

```bash
python magma.py decrypt ecb output.txt decoded.txt ffeeddccbbaa99887766554433221100f0f1f2f3f4f5f6f7f8f9fafbfcfdfeff
```

### Шифрование в режиме CBC

```bash
python magma.py encrypt cbc input.txt output.txt ffeeddccbbaa99887766554433221100f0f1f2f3f4f5f6f7f8f9fafbfcfdfeff 1122334455667788
```

### Расшифрование в режиме CBC

```bash
python magma.py decrypt cbc output.txt decoded.txt ffeeddccbbaa99887766554433221100f0f1f2f3f4f5f6f7f8f9fafbfcfdfeff 1122334455667788
```

## Формат входных и выходных данных

### При зашифровании

* на вход подаётся **любой файл**;
* программа читает его как бинарные данные;
* затем данные отображаются в консоли как текст и как `hex`;
* зашифрованный результат сохраняется в файл в виде `hex`-строки.

### При расшифровании

* на вход подаётся файл с `hex`-данными;
* программа преобразует `hex` обратно в байты;
* расшифрованный результат сохраняется как исходные байты.

## Примечания

* Для режима `CBC` обязательно указывать `IV`.
* В режиме `ECB` параметр `IV` игнорируется.
* Если длина данных не кратна 8 байтам, автоматически применяется `PKCS#7 padding`.
* При расшифровании padding удаляется автоматически, если он корректен.

## Пример работы

Программа выводит в консоль:

1. содержимое входного файла в текстовом виде;
2. содержимое входного файла в hex;
3. зашифрованные данные в hex и в текстовом виде;
4. результат расшифрования.

## Используемый алгоритм

Реализован симметричный блочный шифр **«Магма»** в соответствии с ГОСТ Р 34.12-2015.

Основные параметры:

* размер блока — **64 бита**;
* длина ключа — **256 бит**;
* число раундов — **32**.
